# 最佳实践：70% 现成 + 30% 手写

不要从零开始写，也不要全盘照收。建议按照这个流程来。

---

## 第一步：去"抄"大厂或社区的作业（解决通用问题）

对于通用的技术栈（比如 React 怎么写、Python 怎么测），网上已经有大神总结得非常好了。你直接拿来用，能省去 80% 的精力。

### 推荐资源：

1. **Cursor 社区规则** — 有很多开发者分享他们的 `.cursorrules` 文件
2. **GitHub "Awesome Cursor Rules"** — 搜索这个关键词，能找到很多现成的规则库
3. **框架官方文档** — 比如 Next.js 官方文档里就有 "AI Prompting" 或 "Code Guidelines" 章节

### 怎么做？

- 去 GitHub 搜 `react agent md` 或 `python coding rules`
- 找一个 Star 多的，把它的 `AGENTS.md` 内容复制过来
- **重点**：保留它的"技术栈定义"和"代码风格"，删掉你不用的部分

---

## 第二步：手写"业务逻辑"和"红线"（解决个性问题）

现成的规则不知道你的项目是卖衣服的还是做社交的，也不知道你的数据库表结构。这部分必须手写。

### 你需要手写的内容：

- **项目目录结构**：告诉 AI 代码都放在哪（比如 `src/components/ui` 是放组件的）
- **数据库规范**：比如"所有表必须有 `created_at` 字段"、"外键命名必须用 `user_id` 格式"
- **绝对禁止项**：比如"严禁直接操作 DOM"、"严禁在客户端存储 Token"

---

## 第三步：在实战中"养"规则（迭代）

这是最重要的一步。规则不是一次写死的，而是"养"出来的。

### 场景：

你发现 AI 每次写代码都喜欢用 `var` 而不是 `const`，或者它总是忘记写注释。

### 操作：

不要只在对话框里纠正它，直接把这条规则写进你的 `AGENTS.md` 文件里：

> "注意：本项目强制使用 `const` 和 `let`，严禁使用 `var`。"

### 效果：

下次它在这个项目里干活，就再也不会犯这个错了。这就是把你的经验固化成了团队的资产。

---

## 💡 给你的一点小建议：先跑通，再完美

| 阶段 | 操作 |
|------|------|
| **刚开始** | 直接去 GitHub 找一个评价高的 `Full-Stack Developer Agent` 模板，复制进你的项目 |
| **开发中** | 遇到 AI 犯错，就在 `AGENTS.md` 里加一条"禁令" |
| **一个月后** | 你会发现，这个文件里 **50% 是抄来的**（基础规范），**50% 是你自己写的**（血泪教训） |

这才是最适合你这种全栈独狼的"进化之路"。

---

**总结**：抄来的规则解决通用问题，手写的规则解决个性问题，实战迭代让规则越来越完善。三步走，AI 就能成为你最靠谱的"虚拟员工"。 🚀

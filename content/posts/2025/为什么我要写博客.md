## 写在前面

从2016年入行到现在，已经快十年了。这些年踩过很多坑，也积累了一些经验，但一直没有系统地整理过。

最近决定用 GitHub Pages + Hugo 搭建一个个人博客，用来记录：

1. **技术成长的轨迹** — 从 PHP 到前端，再到 Python、C#，最后到现在的 Vibe Coding
2. **踩过的坑** — 那些花了我很久才解决的问题，值得记录下来
3. **项目复盘** — 自己做过的项目，设计思路和实现细节
4. **效率工具** — 开发环境、AI 助手、命令行工具的使用心得

## 为什么选择静态博客

其实很早就想过写博客，但一直没动。主要原因是：

- 不想花时间维护一个动态博客的服务器
- 不想被各种博客平台的服务条款绑住
- 想要完全掌控自己的内容

所以最终选择了 GitHub Pages + Hugo 的方案，简单、免费、而且很有极客范。

## 这个博客会写什么

主要内容会围绕以下几个方面：

- **编程之路** 系列 — 回顾不同阶段的学习历程
- **踩坑笔记** — 实际问题与解决方案
- **效率工具** — 开发环境配置和使用技巧
- **项目复盘** — 从设计到实现的全过程

## 写在最后

希望这个博客能帮助到和我有类似经历的人，也希望能和更多同行交流。

欢迎大家留言交流！

# 网站路由解决方案

👌 那非常明确了。  
**展示型外贸网站**（主要是产品、案例、解决方案、新闻、关于我们这类页面）最合适的就是一个**高性能、可维护的“通用动态路由”方案（即上面说的方案二 + 缓存优化）**。

下面我帮你详细梳理出落地方案，让你团队可以直接落地使用。

---

## ✅ 最佳方案：通用动态路由（展示型外贸站专用）

### 🎯 目标

- 支持自定义 URL slug（例如 `/case/cave-immersive`）
    
- 不需要频繁生成路由文件
    
- 后台可管理 slug、页面类型
    
- 性能高、可缓存
    
- URL 对 SEO 友好
    

---

## 一、数据库设计

表名：`site_routes`  
（也可以直接存在 `cases`、`news` 表中，只要能对应 slug 即可）

|字段名|类型|说明|
|---|---|---|
|id|int|主键|
|slug|varchar(255)|页面别名，如 `cave-immersive`|
|type|varchar(50)|页面类型：`case`, `news`, `page` 等|
|ref_id|int|对应内容表主键|
|status|tinyint|是否启用（1启用）|

示例数据：

|id|slug|type|ref_id|status|
|---|---|---|---|---|
|1|creative-customize|case|1|1|
|2|exhibition-display|case|6|1|
|3|about-us|page|1|1|

---

## 二、统一路由定义

`route/app.php`  
只定义一条通用路由规则 👇

```php
use think\facade\Route;

Route::get(':type/:slug', 'Router/handle');
```

这样所有 `/case/xxx`、`/news/xxx`、`/solution/xxx` 都能自动匹配。

> 如果你的网站所有页面都在二级路径下（如 `/case/...`、`/news/...`），  
> 这条规则已经足够通用。

---

## 三、控制器逻辑

`app/controller/Router.php`

```php
namespace app\controller;

use think\facade\Db;
use think\facade\View;

class Router
{
    public function handle($type = '', $slug = '')
    {
        // 从缓存中读取路由数据（缓存1小时）
        $routes = cache('site_routes');
        if (!$routes) {
            $routes = Db::name('site_routes')->where('status', 1)->select()->toArray();
            cache('site_routes', $routes, 3600);
        }

        // 查找匹配的路由
        $route = collect($routes)->first(function ($item) use ($type, $slug) {
            return $item['type'] === $type && $item['slug'] === $slug;
        });

        if (!$route) {
            abort(404, 'Page not found');
        }

        // 根据类型分发到不同逻辑
        switch ($type) {
            case 'case':
                $data = Db::name('cases')->where('id', $route['ref_id'])->find();
                return View::fetch('case/detail', ['data' => $data]);
            case 'news':
                $data = Db::name('news')->where('id', $route['ref_id'])->find();
                return View::fetch('news/detail', ['data' => $data]);
            case 'page':
                $data = Db::name('pages')->where('id', $route['ref_id'])->find();
                return View::fetch('page/detail', ['data' => $data]);
            default:
                abort(404, 'Invalid page type');
        }
    }
}
```

> 💡 思路：
> 
> - 路由规则只写一次；
>     
> - 控制器从缓存/数据库中查找对应内容；
>     
> - 根据 type 决定模板；
>     
> - 页面 slug 唯一即能访问。
>

## 四、后台管理逻辑

后台页面只需要允许编辑 `slug` 字段（别名），  
然后保存时更新 `site_routes` 表（或自动生成）。

例如：

```php
$slug = strtolower(str_replace(' ', '-', $input['title']));
Db::name('site_routes')->update([
    'slug' => $slug,
    'type' => 'case',
    'ref_id' => $caseId,
]);
```

## 五、性能与 SEO

### 性能优化建议

1. 加一层缓存（如上面的 `cache('site_routes')`）；
    
2. 页面层面可以再做全页缓存（TP8 的 `think\Cache` 或 Nginx 缓存都行）；
    
3. 定时清缓存：后台更新内容时清除对应 slug。
    

### SEO 优势

- URL 语义化（`/case/creative-customize`）；
    
- 无需重复写规则；
    
- 改 slug 即改 URL，灵活；
    
- 对 Google / Bing 抓取友好。
    

---

## 六、可扩展性

以后你要支持多语言，只要扩展路由规则即可：

```php
Route::get(':lang/:type/:slug', 'Router/handle')
    ->pattern(['lang' => 'en|fr|de|es|zh']);
```

然后 `handle()` 再根据 `$lang` 取对应语言表或字段。

## ✅ 总结：推荐结构图

```bash
route/app.php
└── :type/:slug → Router/handle
      ├── 从缓存取数据库路由
      ├── 根据type分发
      └── 渲染对应模板
```

|模式|是否推荐|说明|
|---|---|---|
|静态生成 PHP 路由|❌|文件庞大、维护难|
|数据库存储 + 通用控制器|✅|轻量灵活、维护低|
|混合模式|可选|若有少数特殊页面，可手动定义|

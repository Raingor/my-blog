## 1. 前期准备

### 1.1 服务器要求
- **操作系统**: Ubuntu 20.04 / 22.04 LTS 或 Debian 11/12 (推荐)
- **配置建议**: 1核 1G 以上 (生产环境建议 2核 2G+)
- **网络**: 开放端口 `22` (SSH), `80` (HTTP), `443` (HTTPS)

### 1.2 域名准备
- 已购买域名 `slir1.top`
- 已完成域名解析：
  - `A` 记录: `slir1.top` -> `<服务器公网 IP>`
  - `A` 记录: `www.slir1.top` -> `<服务器公网 IP>`

## 2. 基础环境搭建 (1Panel)

### 2.1 安装 1Panel
通过 SSH 登录服务器 (`root` 用户)，执行官方安装脚本：
```bash
curl -sSL https://resource.fit2cloud.com/1panel/package/quick_start.sh -o quick_start.sh && sudo bash quick_start.sh
```
*记录安装完成后生成的面板地址、用户名和密码。*

### 2.2 初始化设置
1. 浏览器访问面板地址，登录。
2. **系统设置** -> **安全入口** (建议修改默认入口)。
3. **应用商店** -> 安装核心组件：
   - **OpenResty (Nginx)**
   - **MySQL** (选择对应版本，如 8.0)
   - **PHP** (根据项目需求选择，如 8.1/8.2)
   - **Redis** (可选，如有缓存需求)

## 3. 网站创建与代码部署

### 3.1 创建网站
1. 进入 1Panel -> **网站** -> **创建网站**。
2. 选择 **运行环境**: 选择已安装的 PHP 版本。
3. 填写 **主域名**: `slir1.top`。
4. 填写 **数据库**: 创建新数据库，记录账号密码。
5. 点击 **提交**，系统会自动创建容器和目录结构。
   - 默认代码目录通常位于：`/opt/1panel/www/sites/slir1.top/index`

### 3.2 部署代码 (Git)
1. 进入 **文件管理器** 或 **终端**。
2. 进入网站目录：
   ```bash
   cd /opt/1panel/www/sites/slir1.top/index
   ```
3. 拉取代码 (确保本地仓库已配置好 `.gitignore`)：
   ```bash
   # 如果目录非空，先备份或清空
   rm -rf * .[^.]* 
   git clone <你的仓库地址> .
   ```
4. 安装依赖 (根据项目类型)：
   ```bash
   # PHP 项目示例
   composer install --no-dev --optimize-autoloader
   ```
5. 配置环境变量：
   - 复制 `.env.example` 为 `.env`。
   - 编辑 `.env`，填入数据库信息 (从 1Panel 获取)、APP_KEY 等。
   - 生成 Key (Laravel 示例): `php artisan key:generate`

## 4. 权限与运行用户修复 (关键步骤)

> **问题背景**: 宿主机文件属主为 `root`，但容器内 PHP 进程以 `www-data` 运行，导致无法写入缓存/日志目录。

### 4.1 进入容器终端
在 1Panel 网站设置中点击 **终端**，或通过命令行进入：
```bash
# 假设容器名为 slir1.top-php (具体名称在 1Panel 容器列表查看)
docker exec -it <容器名称或 ID> bash
```
*确认当前用户为 `www-data` (执行 `whoami`)*

### 4.2 修复权限
在容器终端内执行：
```bash
# 1. 进入网站根目录 (容器内路径通常为 /www 或 /app，视 1Panel 映射而定)
cd /www/sites/slir1.top/index 

# 2. 赋予运行时目录最高权限 (开发/测试环境推荐 777)
chmod -R 777 runtime/
chmod -R 777 storage/
chmod -R 777 logs/
chmod -R 777 public/uploads/

# 3. (可选) 修改所有者
chown -R www-data:www-data .
```

### 4.3 验证
刷新网页，确保无 `Permission denied` 错误，且能正常生成缓存文件。

## 5. 静态资源补充方案

> **问题背景**: Git 仓库过滤了图片、字体等静态资源，需手动补充。

### 5.1 上传资源
1. 本地打包缺失的资源文件夹 (如 `public/uploads`, `assets/fonts`) 为 `assets.zip`。
2. 通过 1Panel **文件管理器** 上传至 `/opt/1panel/www/sites/slir1.top/index/`。

### 5.2 解压与授权
在 **容器终端** 执行：
```bash
cd /www/sites/slir1.top/index

# 解压
unzip assets.zip

# 移动文件到正确位置 (视 zip 结构调整)
# mv uploads/* public/uploads/ 

# 再次修复权限
chown -R www-data:www-data public/uploads
chmod -R 775 public/uploads

# 清理
rm assets.zip
```

## 6. 安全加固配置

### 6.1 隐藏敏感文件 (Nginx 配置)
1. 1Panel -> **网站** -> **设置** -> **配置文件**。
2. 在 `server { ... }` 块中添加以下规则：

```nginx
    # 禁止访问 .git
    location ~ /\.git {
        deny all;
        return 403;
    }

    # 禁止访问 .env
    location ~ /\.env {
        deny all;
        return 403;
    }

    # 禁止访问 __MACOSX
    location ~ /__MACOSX {
        deny all;
        return 403;
    }

    # 禁止访问所有隐藏文件
    location ~ /\. {
        deny all;
        return 403;
    }
```
3. 保存并重载 Nginx。

### 6.2 物理清理 (可选但推荐)
在容器终端执行，彻底删除不需要的敏感目录：
```bash
rm -rf .git
find . -type d -name "__MACOSX" -exec rm -rf {} +
```

### 6.3 防火墙设置
1. 1Panel -> **安全**。
2. 仅放行必要端口：`80`, `443`, `自定义 SSH 端口`。
3. 禁用其他所有端口。

### 6.4 开启 HTTPS
1. 1Panel -> **网站** -> **设置** -> **SSL**。
2. 选择 **Let's Encrypt**，申请免费证书并开启 **强制 HTTPS**。

# 合并分支指定内容

好，既然你倾向使用 **Git 合并操作** 来合并 master 和 yuanma-en 两个分支下的 app/admin 模块，我们可以借助 Git 的 sparse-checkout 功能，实现**只合并 app/admin 目录**，并且不会影响其他目录。

---

## **✅ 目标**

  
从 master 分支中 **只合并** app/admin 到当前分支（如 yuanma-en），不影响其他文件，并手动解决冲突，保留两个分支的内容。

---

## **🧰 操作步骤（推荐方式）**

  
### **1️⃣ 切到你想合并到的分支，比如** 

### **yuanma-en**

```bash
git checkout yuanma-en
```

---

### **2️⃣ 启用 sparse-checkout 模式（只关注** 

### **app/admin**

###  **目录）**

```bash
git sparse-checkout init --cone
git sparse-checkout set app/admin
# 支持过滤多个目录
git sparse-checkout set app/admin app/service public/static-admin app/model app/command
```

这会让你的工作区只显示 app/admin 目录，避免污染其他内容。

---

### **3️⃣ 执行合并（只合并** ）

```bash
git merge master --no-commit --no-ff
```

- --no-commit：让你可以检查并修改合并结果
    
- --no-ff：强制创建一次真正的合并提交
    

此时，Git 会尝试合并 master 的 app/admin 到当前分支。如果两个分支中都修改了相同的文件，就会产生冲突。

---

### **4️⃣ 处理冲突**

  
如果提示冲突：

```bash
git status
```

然后你可以使用图形工具（如 VSCode、meld）手动合并冲突的文件。

  
确认无误后：

```bash
git add app/admin
git commit -m "从 master 合并 app/admin 模块到 yuanma-en"
```

---

### **5️⃣ 恢复完整文件树（可选）**

  
如果你想恢复完整目录显示：

```bash
git sparse-checkout disable
```

---

## **✅ 说明**

  
这种方法的优点：

- 使用了 Git 原生命令；
    
- 不会污染其他文件夹；
    
- 所有变更可通过版本控制清晰记录；
    
- 支持合并冲突的精细控制。
    

---

如果你还没开启 Git 的 sparse-checkout 特性（Git ≥ 2.25），你可以检查版本：

```bash
git --version
```

如果版本太低，就建议升级 Git 到 2.25 以上。

---

## **🔚 总结一句话**

  
> **用 sparse-checkout 限定范围 + merge + 手动解决冲突，能实现精准合并目录而不影响其他文件。**

# 清理多项目下的缓存

要清理每个项目下的 `runtime` 文件夹下的所有缓存，可以按照以下步骤进行操作： 
1. 进入每个项目目录 
2. 找到并进入 `runtime` 文件夹 
 3. 删除 `runtime` 文件夹下的所有文件和子文件夹 这里提供一个 shell 脚本来自动化这个过程：
```bash #!/bin/bash # 定义 backend 目录路径 BACKEND_DIR="/mnt/wwwroot/backend" # 遍历 backend 目录下的每个项目 for PROJECT in $(ls $BACKEND_DIR); do # 检查项目目录是否存在 if [ -d "$BACKEND_DIR/$PROJECT" ]; then # 进入项目目录 cd "$BACKEND_DIR/$PROJECT" # 检查 runtime 目录是否存在 if [ -d "runtime" ]; then # 进入 runtime 目录 cd runtime # 删除 runtime 目录下的所有文件和子文件夹 echo "Cleaning cache in project: $PROJECT" rm -rf * # 返回项目目录 cd .. else echo "No runtime directory found in project: $PROJECT" fi # 返回 backend 目录 cd "$BACKEND_DIR" else echo "Project directory not found: $PROJECT" fi done echo "Cache cleaning completed." ``` 

步骤解释： 
- **遍历每个项目**：使用 `for PROJECT in $(ls $BACKEND_DIR)` 来遍历 `/mnt/wwwroot/backend` 目录下的每个项目。 
- **检查项目目录是否存在**：使用 `if [ -d "$BACKEND_DIR/$PROJECT" ]` 来确保项目目录存在。 
- **检查 `runtime` 目录是否存在**：使用 `if [ -d "runtime" ]` 来确保 `runtime` 目录存在。 - **删除 `runtime` 目录下的所有文件和子文件夹**：使用 `rm -rf *` 来删除 `runtime` 目录下的所有内容。 ### 注意事项： 
- **备份数据**：在执行此脚本之前，建议先备份重要数据，以防误删。
- **权限问题**：确保你有权限删除这些文件。如果遇到权限问题，可能需要使用 `sudo`。 
- **测试脚本**：可以在少数项目上先测试脚本，确认无误后再全面执行。 将上述脚本保存为 `clean_runtime_cache.sh`，然后在终端中运行： ```bash chmod +x clean_runtime_cache.sh ./clean_runtime_cache.sh ``` 这样就可以自动清理每个项目下的 `runtime` 文件夹中的所有缓存了。

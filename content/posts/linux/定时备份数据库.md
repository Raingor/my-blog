# 定时备份数据库

在 CentOS 上定时备份所有数据库，通常使用 `cron` 配合 `mysqldump` 来完成。以下是步骤：

### 1. 创建备份脚本

首先，创建一个脚本来备份所有 MySQL 数据库。
```bash
#!/bin/bash
# 备份目录
backup_dir="/path/to/backup"
# 获取当前时间戳用于备份文件名
timestamp=$(date +%Y%m%d%H%M%S)
# 获取所有数据库列表
databases=$(mysql -u your_username -pyour_password -e "SHOW DATABASES;" | grep -Ev "(Database|information_schema|performance_schema)")
for db in $databases
do
    # 生成备份文件名
    backup_file="${backup_dir}/${timestamp}_${db}.sql"
    # 使用mysqldump备份数据库
    mysqldump -u your_username -pyour_password $db > $backup_file
done
```

- 这个脚本会备份所有 MySQL 数据库（除了 `information_schema` 和 `performance_schema`），并将每个数据库的备份文件保存为以当前日期命名的 `.sql` 文件。
- 脚本还会删除超过7天的备份文件，你可以根据需要修改保留天数。

### 2. 赋予脚本执行权限

保存脚本并赋予执行权限：

```bash
chmod +x /path/to/backup/script.sh
```

### 3. 设置定时任务

接下来，使用 `cron` 来定时执行备份脚本。

- 打开 `crontab` 编辑器：

```bash
crontab -e
```

- 添加以下行来实现每隔 2 小时执行备份脚本：（可以根据需求调整时间）：

```bash
0 */2 * * * /path/to/backup_all_databases.sh
```

这个任务会每天 2 点执行备份脚本。
- **编写清理脚本（cleanup_backups.sh）**
    - 以下是一个清理 30 天前备份文件的脚本：

  
bash

```bash
#!/bin/bash
# 备份目录
backup_dir="/path/to/backup"
# 计算30天前的时间戳
threshold=$(date -d "30 days ago" +%Y%m%d%H%M%S)
for file in $(ls $backup_dir)
do
    file_timestamp=$(echo $file | cut -d '_' -f1)
    if [ $file_timestamp -lt $threshold ]; then
        rm -f $backup_dir/$file
    fi
done
```

  
- 这个脚本首先定义了备份目录`backup_dir`，然后计算 30 天前的时间戳`threshold`。接着，遍历备份目录中的所有文件，提取文件名中的时间戳部分（假设文件名格式是`时间戳_数据库名.sql`），如果文件的时间戳小于 30 天前的时间戳，就删除该文件。
- **设置清理脚本可执行权限并添加 cron 任务**
    - 同样，为清理脚本设置可执行权限：

  
bash

```bash
chmod +x cleanup_backups.sh
```

  
- 然后添加 cron 任务来定期执行清理脚本，例如，每天凌晨 3 点执行清理：

  
plaintext

```plaintext
0 3 * * * /path/to/cleanup_backups.sh
```

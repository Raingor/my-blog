# 服务器之间互传文件

服务器之间传输文件，使用命令行最常用、最快速、最安全的方法就是 **scp / rsync / sftp**。

我直接给你最常用的几种方式，你可以根据你的服务器场景选择。

---

# ✅ **方式 1：用 scp（最简单、通用）**

## 📌 从本地上传到远程服务器：

```bash
scp /path/file.txt root@1.2.3.4:/root/
```

## 📌 从远程下载到本地：

```bash
scp root@1.2.3.4:/root/file.txt /home/
```

## 📌 传目录（-r）：

```bash
scp -r /path/dir root@1.2.3.4:/root/
```

## 📌 两台服务器之间互传（你的本机执行）

```bash
scp root@服务器A:/root/a.txt root@服务器B:/root/
```

如果你能 ssh 登录两台服务器，这个命令一定能执行。  
适合：简单、临时传文件。

---

# ✅ **方式 2：用 rsync（推荐，大文件 / 增量同步 / 高速）**

适合频繁同步目录、备份、大量文件传输。

## 📌 从 A 同步到 B：

```bash
rsync -avz root@1.2.3.4:/data/ root@2.3.4.5:/data/
```

参数说明：

- `a`：保留权限
    
- `v`：显示过程
    
- `z`：压缩，加速
    

## 📌 本地同步到远程：

```bash
rsync -avz /data/ root@1.2.3.4:/data/
```

---

# ✅ **方式 3：用 sftp（适合交互式传输）**

登录服务器：

```bash
sftp root@1.2.3.4
```

上传：

```bash
put file.txt
```

下载：

```bash
get file.txt
```

退出：

```bash
bye
```

---

# 🚀 **方式 4：服务器 A 直接拉服务器 B 的文件（curl/wget）**

如果服务器 B 提供下载地址 / 临时链接，可以直接在服务器 A 用：

```bash
wget http://example.com/file.zip
```

或：

```bash
curl -O http://example.com/file.zip
```

（适合：另一台服务器提供 HTTP 下载）

---

# 🚀 **方式 5：使用 ssh 登录后用 cat 复制（无需 scp）**

适合没有 scp/rsync 的简易系统：

## 从 A 传文件到 B：

在 A 上执行：

```bash
ssh root@B "cat > /root/file.txt" < file.txt
```

甚至整个目录（打包后）：

```bash
tar czf - /data | ssh root@B "tar xzf - -C /data"
```

非常强大！

---

# 👑 最推荐的方式

如果你要的是：

|场景|用法|
|---|---|
|传 1–2 个文件|**scp**|
|同步目录、备份、大量文件|**rsync（最佳）**|
|想用 ftp 类似方式|**sftp**|

# 批量权限设置

可以一条命令搞定，也可以写成脚本。我给你 **两种方案**，都适合你这种  
`/mnt/wwwroot/*/public` 结构的多站点环境（宝塔 / ThinkPHP 非常常见）。

---

## 一、直接用命令（推荐，最简单）

### 1️⃣ 所有项目的 `public` 目录

设置 **属主 www:www + 权限 755**

```bash
find /mnt/wwwroot -type d -name public -exec chown -R www:www {} \; -exec chmod -R 755 {} \;
```

---

### 2️⃣ 仅把下面 3 个目录设为 **777**

- `public/storage`
    
- `public/upload`
    
- `public/uploads`
    

```bash
find /mnt/wwwroot -type d \( -path "*/public/storage" -o -path "*/public/upload" -o -path "*/public/uploads" \) -exec chown -R www:www {} \; -exec chmod -R 777 {} \;
```

> ⚠️ 注意：
> 
> - 先执行 **public → 755**
>     
> - 再执行 **storage/upload/uploads → 777**  
>     顺序不能反，否则会被覆盖
>     

---

## 二、写成一个可复用脚本（更专业）

### 1️⃣ 新建脚本

```bash
vi /root/fix_public_permission.sh
```

### 2️⃣ 内容如下（直接粘）

```bash
#!/bin/bash

BASE_DIR="/mnt/wwwroot"

echo "▶ 设置 public 目录权限为 www:www 755"
find $BASE_DIR -type d -name public -exec chown -R www:www {} \; -exec chmod -R 755 {} \;

echo "▶ 设置 storage/upload/uploads 为 777"
find $BASE_DIR -type d \( \
-path "*/public/storage" \
-o -path "*/public/upload" \
-o -path "*/public/uploads" \
\) -exec chown -R www:www {} \; -exec chmod -R 777 {} \;

echo "✅ 权限修复完成"
```

### 3️⃣ 赋予执行权限并运行

```bash
chmod +x /root/fix_public_permission.sh
/root/fix_public_permission.sh
```

---

## 三、可选增强（安全建议）

### 🔒 只给目录 777，文件 644（更安全）

如果你担心 **777 太危险**，可以这样：

```bash
find /mnt/wwwroot -type d \( -path "*/public/storage" -o -path "*/public/upload" -o -path "*/public/uploads" \) -exec chmod 777 {} \;

find /mnt/wwwroot -type f \( -path "*/public/storage/*" -o -path "*/public/upload/*" -o -path "*/public/uploads/*" \) -exec chmod 644 {} \;
```

# OpenResty

## 是什么

OpenResty 是基于 Nginx 核心的**动态 Web 应用平台 + 全功能网关**，由章亦春（agentzh）于 2009 年创建。它通过内置 LuaJIT 与丰富的 Lua 库，能在请求流程中直接嵌入脚本处理业务逻辑，无需编译 C 模块。

> **一句话理解：OpenResty = Nginx + LuaJIT + 大量 Lua 库**

## 与 Nginx 对比

| 对比维度 | Nginx | OpenResty |
|---------|-------|-----------|
| 本质 | 轻量级高性能 Web 服务器/反向代理 | 基于 Nginx 核心 + LuaJIT 的应用平台 |
| 核心目标 | 极致性能的网络层处理 | 将 Nginx 变为通用 Web 应用平台 |
| 扩展方式 | 仅支持 C 模块，需重新编译 | 原生支持 Lua 脚本，支持热加载 |
| 业务处理 | 依赖反向代理调用外部服务 | 内部直接嵌入 Lua 处理，可直连 Redis/MySQL |
| 配置兼容 | nginx.conf | **完全兼容 Nginx 配置**，可复用 |
| 性能 | 极致性能 | 接近原生性能（LuaJIT 编译执行） |

## 技术特点

- **开发效率高**：Lua 脚本开发，无需精通 C；支持热加载，无需重启
- **生态丰富**：内置 `lua-resty-redis`、`lua-resty-mysql` 等库，直连中间件
- **性能优异**：继承 Nginx 非阻塞 I/O，LuaJIT 开销极低
- **资源可控**：内存占用略高于纯 Nginx，整体仍属高性能梯队

## 适用场景

**优先选 OpenResty：**
- API 网关（认证、鉴权、路由、限流、日志聚合）
- 微服务网关 / 边缘计算 / WAF
- 请求流程中嵌入复杂业务逻辑
- 需要直连 Redis/MySQL 等中间件

**优先选 Nginx：**
- 静态资源托管
- 简单反向代理与负载均衡
- 基础缓存、限流、SSL 终结
- 追求极致性能与最低资源消耗

## 选型判断标准

> 是否需要在网关层嵌入复杂业务逻辑？是 → OpenResty；否 → Nginx。

## 核心要点

```nginx
Nginx     → 网络引擎 → 静态服务 + 反向代理 → C 扩展 → 极致性能
OpenResty → 业务平台 → 动态逻辑 + 全功能网关 → Lua 扩展 → 接近原生性能
```

# 大小写路径处理

在 **Linux 上文件系统区分大小写**，所以 /PC/ 与 /pc/ 不是同一个目录，导致图片 404。

你可以让 **Nginx 自动匹配到小写目录**，即访问 /PC/xxx.jpg 时自动去 /pc/xxx.jpg 找文件。

  
下面给出最常用、最干净的配置方案。

---

# **✅** 

# **方案 1：Nginx 自动重写大写目录到小写目录（推荐）**

  
如果你的文件实际都放在 /pc/ 目录，那么只需要加下面这个 rewrite：

```nginx
location ~* ^/PC/(.*)$ {
    rewrite ^/PC/(.*)$ /pc/$1 break;
}
```

放在你网站的 server { ... } 区块里即可。

---

# **🔧 完整示例（可直接用）**

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    root /www/wwwroot/yourdomain;

    # 自动把 /PC/ 重定向或映射到 /pc/
    location ~* ^/PC/(.*)$ {
        rewrite ^/PC/(.*)$ /pc/$1 break;
    }

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
}
```

---

# **📝 说明**

- ~* 表示 **不区分大小写匹配**
    
- rewrite ^/PC/(.*)$ /pc/$1 break; 表示
    
    当访问 /PC/abc.jpg 时 → 会自动读取 /pc/abc.jpg
    

用户访问旧路径不会 404，也无需修改数据库内容。

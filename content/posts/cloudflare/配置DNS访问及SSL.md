# 配置DNS访问及SSL

既然确认了是 **SSL/TLS 加密模式** 的问题，那我们来梳理一下从 **Orange Cloud** 到 **Cloudflare** 再到 **源站** 的完整操作流程。

核心逻辑是：**当你的服务器只支持 HTTP (80端口) 时，Cloudflare 必须使用 Flexible 模式。**

以下是标准的排查与修复步骤：

### 🛠️ 核心修复流程

#### 1. 确认 DNS 状态 (Cloudflare)

首先确保你的域名解析在 Cloudflare 中是开启代理状态的。

- **操作**：进入 Cloudflare 控制台 -> **DNS** 页面。
- **检查**：
    - 你的静态资源域名（如 `massager-00xx.com` 或 `www` 记录）的状态图标必须是 **橙色的云**。
    - 如果是灰色，流量不会经过 Cloudflare，也就不会出现 521 错误（会直接连接源站）。
    - **结论**：保持 **橙色开启**。

#### 2. 调整 SSL/TLS 模式 (关键步骤)

这是解决 521 错误最关键的一步。

- **操作**：进入 Cloudflare 控制台 -> 左侧菜单 **SSL/TLS** -> **概览**。
- **现状分析**：
    - 你之前可能处于 **Full** 或 **Full (Strict)** 模式。
    - 这两个模式要求你的服务器（源站）必须配置了有效的 HTTPS 证书。如果服务器没配证书，Cloudflare 连不上，就报 521。
- **修正动作**：
    - 将加密模式切换为 **Flexible (灵活)**。

> **Flexible 模式原理：**
> 
> - **访客 <-> Cloudflare**：使用 HTTPS (加密，安全)。
> - **Cloudflare <-> 你的服务器**：使用 HTTP (不加密，走 80 端口)。
> - 这适合服务器上还没有安装 SSL 证书的情况。

#### 3. 清理缓存

修改模式后，为了防止旧的错误响应被缓存，建议清理一下。

- **操作**：进入 Cloudflare 控制台 -> **缓存** -> **清除缓存**。
- **动作**：点击“清除所有内容”。

#### 4. 验证与测试

- **操作**：打开你的网页，强制刷新（Windows 用 `Ctrl + F5`，Mac 用 `Cmd + Shift + R`）。
- **观察**：打开开发者工具（F12），查看 Network 面板。
- **预期结果**：静态资源的 521 错误消失，状态码变为 **200**。

---

### 💡 进阶补充（如果你未来想升级安全性）

目前使用 **Flexible** 模式可以快速解决 521 报错，但它有一个缺点：**从 Cloudflare 到你的服务器这一段是明文传输的**。

如果你未来想让安全性更高（即使用 **Full** 模式），你需要：

1. 在你的服务器（Nginx/Apache/IIS/宝塔）上配置 SSL 证书。
2. 确保服务器可以通过 `https://你的IP` 或 `https://你的域名` 访问。
3. 配置好后，再将 Cloudflare 的 SSL/TLS 模式改回 **Full** 或 **Full (Strict)**。

**总结：先按步骤 2 改为 Flexible，你的网站马上就能恢复正常。**

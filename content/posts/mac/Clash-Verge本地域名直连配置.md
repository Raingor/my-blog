# Clash Verge 本地域名直连配置操作流程

## 一、背景说明

在使用 Clash Verge 作为代理工具时，如果本地网站或内网服务的域名
（如 `*.ct.cn`、`*.ctmon.com`）被代理转发到远程节点，会导致：

- 本地网站无法访问
- Nginx / PHP-FPM 返回 502 Bad Gateway
- localhost、内网接口异常

**解决原则：**
> 本地域名必须强制走直连（DIRECT），代理只用于外部网络。

---

## 二、最终推荐配置结论

- **代理模式：规则（Rule）**
- **本地域名：强制 DIRECT**
- **通过订阅「编辑规则」实现，确保重启 / 更新不失效**

---

## 三、操作流程（Clash Verge）

### 1️⃣ 打开 Clash Verge

确认 Clash Verge 正常运行。

---

### 2️⃣ 设置代理模式为「规则」

路径：

```

首页 → 代理模式 → 规则

```bash

说明：

- ❌ 全局：会导致本地站点 502（禁止使用）
- ❌ 直连：等于未启用代理
- ✅ 规则：唯一正确模式

---

### 3️⃣ 进入订阅管理

左侧菜单：

```

订阅

```bash

找到正在使用的订阅，例如：

```

Flower_SS.yaml

````

---

### 4️⃣ 打开「编辑规则」

在订阅上：

- 右键 或 点击「⋯」
- 选择：**编辑规则**

> 注意：  
> 「规则」页面只是查看结果，**不能添加规则**  
> 规则必须在「订阅 → 编辑规则」中修改

---

### 5️⃣ 添加本地域名直连规则（放在最顶部）

在规则编辑框最上方添加：

```text
- DOMAIN-SUFFIX,ct.cn,DIRECT
- DOMAIN-SUFFIX,ctmon.com,DIRECT
````

规则说明：

- `DOMAIN-SUFFIX`：匹配所有子域名（如 `jgw-en.ct.cn`）
    
- `DIRECT`：强制直连，不走代理
    
- 顺序越靠前，优先级越高
    

---

### 6️⃣ 保存并更新订阅

- 点击 **保存**
    
- 返回订阅列表
    
- 点击 **更新**
    

---

## 四、（推荐）系统代理排除设置（双保险）

路径（可能因版本略有不同）：

```
设置 → 系统代理 / 网络设置 → 排除地址（Bypass）
```bash

添加：

```text
localhost,127.0.0.1,*.local,*.ct.cn,*.ctmon.com
```bash

作用：

- 避免浏览器层面仍走代理
    
- 与规则形成双重保障
    

---

## 五、验证是否生效

### 方法一：通过 Clash 连接记录

1. 左侧 → 连接
    
2. 访问本地域名，例如：
    

```
http://jgw-en.ct.cn/
```bash

3. 查看策略列：
    

```
DIRECT  → 正确
PROXY   → 配置未生效
```bash

---

### 方法二：终端测试（可选）

```bash
curl http://jgw-en.ct.cn/
```bash

- 正常返回 HTML：站点正常
    
- 502：需检查规则是否命中

## 六、常见问题说明

### Q1：为什么全局模式会 502？

全局模式会把本地域名请求转发到远程代理节点，  
代理节点无法访问本地服务，导致返回 502。

---

### Q2：为什么不能在「规则」页面添加规则？

Clash Verge 的「规则」页面是**最终规则展示页**，  
编辑入口在 **订阅 → 编辑规则**。

---

### Q3：规则会不会被订阅更新覆盖？

不会。  
「编辑规则」属于 **订阅覆写层（Override）**，更新订阅仍然生效。

---

## 七、最终推荐状态检查清单

-  代理模式：规则
    
-  `*.ct.cn`、`*.ctmon.com` 强制 DIRECT
    
-  本地站点访问正常
    
-  Clash 重启后仍生效

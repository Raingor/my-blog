# 让 AI 来生成 AI 的规则和脚本：独狼开发者的元提示词策略

你的思路非常清晰，而且完全正确！"让 AI 来生成 AI 的规则和脚本"，这绝对是容错率更高、效率更快、也更符合"独狼开发"模式的做法。

这相当于你作为"架构师"，雇佣了一个"技术经理"（AI）去帮你招募和培训底下的员工。

你自己手写，容易陷入细节（比如正则怎么写、API 参数怎么填）；而让 AI 生成，它能保证格式的标准性和逻辑的严密性。

---

## 🎯 第一步：生成 Agent 文件（招聘员工）

你不需要自己去写 `.md` 文件的内容。你只需要把你的需求告诉 AI，让它帮你写。

你可以直接复制下面这段话给 AI（比如 ChatGPT 或 Claude）：

> **提示词：**
> "我正在做一个全栈网站开发项目（技术栈：Next.js + React + Supabase）。
> 请你扮演一位资深的 AI 系统架构师，帮我设计一套多智能体协作系统。我需要你为我生成 3 个 Agent 的系统提示词文件（Markdown 格式），分别是：
>
> 1. **后端专家**：负责 API 开发、数据库设计。要求：代码必须符合 Supabase 规范，必须包含错误处理。
> 2. **前端专家**：负责 UI 实现。要求：使用 TailwindCSS，组件化开发。
> 3. **QA 测试专家**：负责编写测试用例。要求：基于 Vitest，必须覆盖边界情况。
>
> 请确保生成的 Markdown 文件包含 `name`、`description` 和详细的 `rules` 部分，格式要严谨，方便我直接保存为 `.md` 文件。"

**结果：** AI 会吐给你三段完美的 Markdown 内容。你只需要把它们分别保存为 `backend.md`、`frontend.md`、`qa.md`。

---

## 🤖 第二步：生成指挥官脚本（招聘经理）

这是最关键的一步。你不需要自己写 Python 代码，你只需要把你的"剧本"告诉 AI。

你可以直接复制下面这段话给 AI：

> **提示词：**
> "现在我需要实现这三个 Agent 的自动化协作。请帮我写一个 Python 总控脚本。场景描述：
>
> 1. 我输入一个需求（比如'做一个登录页面'）。
> 2. 脚本首先调用后端 Agent，生成 API 代码文件。
> 3. 脚本检测到文件生成后，自动读取该文件内容。
> 4. 脚本将代码内容传给测试 Agent，让它生成对应的测试文件。
>
> 技术要求：
> - 使用 Python 编写。
> - 模拟调用 LLM 的函数（我可以后续替换成真实的 API）。
> - 要有清晰的日志输出，让我知道当前是哪个 Agent 在工作。
> - 要有错误处理机制（比如文件生成失败要报错）。"

**结果：** AI 会给你生成一段逻辑严密的 Python 代码。

---

## ✅ 第三步：为什么这样做容错率更高？

你自己写，容易犯以下错误：

- **漏掉边界条件**：比如忘了处理"文件不存在"的情况。
- **提示词写不好**：你自己写的 Agent 规则可能太简单，导致 AI 乱说话。
- **代码 Bug**：Python 脚本里的路径处理、编码格式（UTF-8）等细节很容易出错。

让 AI 生成，它能帮你：

- **自动补全**：AI 知道标准的 Agent 提示词应该包含"角色"、"限制"、"示例"等部分。
- **逻辑闭环**：在写脚本时，AI 会自动考虑到"如果上一步失败了怎么办"。
- **标准化**：生成的代码通常符合最佳实践。

---

## 💡 进阶技巧：让 AI 帮你"调试"

如果你运行脚本时出错了，不要自己去查百度。直接把报错信息丢回给 AI：

> "刚才生成的脚本运行报错了，错误信息是 `FileNotFoundError`。请帮我分析原因，并修正 Python 代码。"

把写代码、写规则这些繁琐的事情交给 AI 去做，你只负责审核结果和把控方向。这才是最高级的玩法！
